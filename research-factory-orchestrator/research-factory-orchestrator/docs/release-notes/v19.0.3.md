# Release notes — v19.0.3

## SemVer disclaimer (inherits v19.0.2 + behavioral hardening)

v19.0.3 is patch-numbered per cleanup policy, but contains **behavioral contract hardening**:

- **V6** `DELIV-STUB-EXT` semantic restored to spec (external delivery flags vs attachment extension confusion).
- **V6** `DELIV-ATT-MISSING` semantic restored to spec (attachment paths must exist on disk).
- **V6** new issue code **`DELIV-STUB-FORBIDDEN-ATTACHMENT`** (preserves prior extension guard under the correct name).
- **J3** V1 stdlib walker: `oneOf` / `anyOf` / `allOf` semantics corrected (new codes `V1-SCHEMA-ONEOF-NO-MATCH`, `V1-SCHEMA-ONEOF-MULTI-MATCH`, `V1-SCHEMA-ANYOF-NO-MATCH`); `allOf` no longer masks sibling `additionalProperties` enforcement.
- **J6** unified claim statuses via `contracts/claim-statuses.json` (`partial` removed; legacy alias documented).

Consumers should treat v19.0.3 as a **semver-exception** patch for proof-layer behavior. Pinning to `==19.0.2` is reasonable if you depended on the pre-fix V6 / walker behavior; migrating to `==19.0.3` requires re-checking downstream assumptions.
