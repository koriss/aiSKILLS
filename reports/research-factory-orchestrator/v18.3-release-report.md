# RFO v18.3 release report

Version: `18.3.0-source-quality-execution-reliability-hardening`

Base: v18.1.1 full overlay.

Added:

```text
source-quality weighting and E0-E5 evidence classes
source acquisition states and source gap ledger contracts
source acquisition broker skeleton
inference broker skeleton
execution reliability policy
model-call ledger / timeout ledger / partial output protocol
worker lease / heartbeat validators
ambient-agent override guard
cron isolation validator
F170-F182, F200-F209, F210-F229, F230-F263 failure classes
```

This release targets observed operational failures:

```text
LLM idle timeout 120s
provider/runtime request timeout
cron execution timeout
blocked/unavailable upstream source acquisition
subagent timeout and partial-output cases
local model disconnect / context overflow / OOM style failures
```

Design stance:

```text
The skill must not pretend external systems are reliable.
It must record failures, salvage partials safely, cap claims, and avoid false completion.
```
