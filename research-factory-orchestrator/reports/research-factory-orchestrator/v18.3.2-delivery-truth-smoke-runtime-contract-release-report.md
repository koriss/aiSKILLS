# RFO v18.3.2 release report

Release: `18.3.2-delivery-truth-smoke-runtime-contract-hotfix`

## Fixed / added

- Added top-of-SKILL operator protocol: how to use the skill and what user-visible claims are forbidden.
- Added delivery truth guards: no attachment/sent/delivered claims without manifest + ack evidence.
- Added smoke-run guard: seed/smoke scaffolds cannot be presented as production research.
- Added manual fallback guard: manual synthesis cannot be called RFO output unless integrated into ledgers and gates rerun.
- Fixed `validate_runtime.py` contract drift: default profile now matches `init_runtime.py` current v18 layout; legacy profile is explicit.
- Updated `scripts/README.md`: `validate_schemas.py` requires `<schemas_dir>`.
- Added modularization boundary package `runtime/` and `core-boundary-contract.json` for future god-module decomposition.
- Added validation-source-of-truth contract: JSON artifacts are primary; HTML string heuristics are render-hygiene only.

## Known architecture debt

`rfo_v18_core.py` remains a compatibility god module in this hotfix. v18.3.2 adds boundaries and validators but does not pretend to complete the full refactor. Full decomposition should be v18.4/v19.
