# RFO 18.3.1-context-integrity-hotfix Release Report

Full-overlay hotfix over v18.3.

## Added

- Context Acquisition Integrity Layer.
- Mechanical distinction between read/index/load/execute/smoke/memory-write.
- New context contract, policy, schemas, validators, templates, and failure corpus F270-F289.
- Guards against false “I read all files” / “loaded into context” / smoke-test-as-context-proof / subagent-started-as-completed / memory-write-as-active-context.
- Chat payload reasoning leak validator.

## Why

The observed Telegram log showed repeated self-certification after incomplete reads, misuse of subagent for 673 files, smoke-test being described as context loading, and reasoning/debug leakage to chat.

## Status

This release hardens the skill against the observed failure class. It does not promise that host agents can literally keep a 600+ file workspace in active context. It forces honest operational-core + indexed/on-demand modes unless ledgers prove otherwise.
