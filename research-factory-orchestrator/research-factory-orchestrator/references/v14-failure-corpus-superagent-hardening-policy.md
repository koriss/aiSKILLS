
# v14 Failure-Corpus + Super-Agent Hardening Policy

The skill must learn from failure classes, not from a single agent implementation.

Previous reports are regression examples. Their errors are generic agent-network risks:
- timeout cascades;
- partial output masquerading as complete;
- declarative proof;
- missing ledgers;
- overconfident negative identity;
- source-family/source-count overclaims;
- local path leaks;
- Telegram tables;
- manual proof;
- inference promoted to fact.

Every future run must produce machine-checkable artifacts instead of prose-only proof.
