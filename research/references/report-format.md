# Формат HTML-отчёта и соответствие deep-investigation-agent

## Совместимость

Структура содержательно **не противоречит** философии deep-investigation-agent: факты, claims, timeline, гипотезы, противоречия, unknowns, confidence, sources.

HTML дополняет это **оркестрационными** блоками: subagent review, execution metadata (кратко), расширенная verification-карта для режима verify.

## Обязательные секции HTML

1. Executive Summary  
2. Главный вопрос и подвопросы  
3. План сбора данных (как собирали)  
4. Facts  
5. Claims / Verification (в verify — атомарные утверждения и статусы)  
6. Evidence Map  
7. Timeline  
8. Actors  
9. Причинная цепочка  
10. Contradictions  
11. Hypotheses (минимум 3) + stress-test лучшей  
12. Confidence  
13. Unknowns  
14. Subagent Review  
15. Sources  

## Статусы утверждений (режим verify)

Используй единый набор:

| Статус | Смысл |
|--------|--------|
| `confirmed` | подтверждено сильными источниками |
| `false` | опровергнуто |
| `distorted` | искажено / вводит в заблуждение |
| `unverified` | нет достаточных доказательств |
| `out_of_context` | вне контекста / неверная подстановка |
| `partially_true` | частично верно |

## Иерархия источников

1. Первичные: документы, регистры, стенограммы, filings, датасеты, статьи с данными, CVE, коммиты, оригинальные заявления.  
2. Сильная вторичка: обзоры, расследования, экспертные центры.  
3. Tertiary: агрегаторы, пересказы, соцсети — только контекст или lead, не опора для «подтверждено».

Правила: repetition ≠ corroboration; 10 пересказов одной заметки ≠ 10 подтверждений.

## Типографика

Встроенные стили в `templates/report.html`: читаемые шрифты системного стека, нормальные отступы, таблицы для матриц «тезис ↔ источник».
