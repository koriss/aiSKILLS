# RFO v18.3 — Source Quality + Execution Reliability Hardening

Version: `18.3.0-source-quality-execution-reliability-hardening`

This release is a full overlay on top of v18.1.1. It is built to improve stability under unreliable model/provider/tool conditions, not to "fix one agent".

## Added reliability layers

```text
source quality weighting
source acquisition reliability
execution reliability
inference broker contract
partial output protocol
worker lease / heartbeat contracts
final gate caps for timeout/partial/snippet-only evidence
```

## Core invariants

```text
Timeout is data, not invisible failure.
Partial output is not final.
Retry must change something.
Snippet-only evidence cannot confirm a claim.
Local model disconnect must not kill the whole research run.
Official source is authoritative only inside scope.
Political/blog/opinion sources can be useful, but must be role-scoped.
```

## New gates

```text
source_quality_gate
source_acquisition_gate
execution_reliability_gate
model_call_gate
partial_output_gate
```

## Added failure ranges

```text
F170-F182 source quality / role-fit / propaganda nuance
F200-F209 source acquisition / snippet / blocked source handling
F210-F229 model/subagent timeout / partial output / local backend reliability
F230-F233 ambient-agent override / direct subagent bypass
F240-F263 lifecycle / retry / cron isolation
```
