# Release notes — v19.0.2

## SemVer / patch policy (post-release documentation note)

Technically **v19.0.x** may ship **behavioral** validator or contract hardening while staying patch-numbered (cleanup policy). Consumers pinning `~=19.0` should read each patch’s release notes. Breaking renames (e.g. KB field rename below) are called out explicitly under **BREAKING**.

## CRITICAL
- Added strict stdlib schema walking in V1 with explicit unsupported keyword detection.
- Added fixture-hygiene enforcement and expected-driven suite runners for v19 and release_bad fixtures.
- Added delivery safety checks: attachment presence/path/stub-extension constraints.

## BREAKING
- `kb_as_factual_source` renamed to `kb_match_used_as_evidence` (no alias).
- V1 now fails on schema shape drift in core artifacts before downstream validators.
- Status-cap checks are profile-driven via `contracts/status-rank.json`.
