# Orchestration: supervisor-слой над deep-investigation-agent

## Роли

| Роль | Функция |
|------|---------|
| **Planner** | Нормализует вопрос, выбирает режим `investigate` / `verify`, оценивает жирность и риск overflow |
| **Splitter** | Строит декомпозицию и task contracts для subagent-ов |
| **Researcher (sub)** | Собирает доказательства по своему scope; сохраняет артефакт |
| **Critic** | После каждого sub: чеклист, рейтинг, failure_flags, решение dosbor/retry |
| **Synthesizer** | Сводит evidence map, гипотезы, финальный HTML/JSON/Telegram |

## Правило включения multi-agent

Переключайся на orchestration, если выполняется **хотя бы одно**:

- много тезисов или сущностей;
- длинный входной текст;
- несколько конфликтующих объяснений в публичном поле;
- широкий временной диапазон;
- явный риск context overflow (оценочно «большой» контекст задачи).

Оставайся на **single-agent**, если задача узкая, один тезис, мало сущностей и укладывается в один компактный research loop.

## Ограничения

- Одновременно активных subagent-ов: **не более 5**.
- Целевой диапазон: **3–5** subagent-ов.
- Если слот занят и subagent завис — дождаться завершения других, затем **re-split** проблемного фрагмента.

## Стратегии разбиения (порядок применения)

1. По **подвопросам** (наиболее предпочтительно).
2. По **кластерам тезисов** (связанные утверждения вместе).
3. По **типам доказательств** (реестры / СМИ / научные источники / соцсети как слабый слой).
4. По **временным периодам** (до/после ключевого события).
5. По **акторам / сторонам конфликта**.
6. По **конкурирующим гипотезам**.

Splitter обязан кратко зафиксировать в `execution-log.json`: `chosen_strategy` и почему.

## Эвристики по умолчанию

| Параметр | Значение |
|----------|----------|
| Порог «жирной» задачи (слова входа) | > 800 |
| Порог числа атомарных утверждений (verify) | > 8 |
| Порог именованных сущностей | > 6 |
| Временной диапазон «широкий» | > 2 лет между крайними якорями |
| Оценка overflow (эвристика) | много подвопросов + длинный контекст + требование широкого обзора → `high` |

## Сколько subagent-ов: 3, 4 или 5

- **3:** три крупных кластера (например хронология / акторы / гипотезы).
- **4:** добавлен отдельный кластер «источники и методология» или отдельный блок verify.
- **5:** максимум при явной многокомпонентной задаче; не добавлять пятый без причины.

## Дедупликация

- Перед запуском subagent-ов явно развести **непересекающиеся** scope (кроме минимального пересечения для стыковки).
- В synthesizer помечать, если два куска сошлись на одном факте — не считать это двумя независимыми подтверждениями.

## Маппинг intent → mode

- Ключевые маркеры **verify:** проверь, фактчек, правда ли, опровергни, разбей на утверждения.
- Ключевые маркеры **investigate:** исследуй, картина, сценарии, кто выигрывает, как развилось.

При сомнении: если пользователь принёс **текст для разбора** — чаще **verify**; если **открытый вопрос о мире** — **investigate**.
